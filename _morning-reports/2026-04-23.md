## Top 3 Things Worth Paying Attention To

### 1. "Tasteful Tokenmaxxing" — the new mental model replacing "more AI = better"
**What it is:** The top conversation at AI leadership conferences right now (AIE Miami just wrapped) is how to get teams to use *more* AI without incentivizing wasteful sprawl. Shopify CTO Mikhail Parakhin (in a new Latent Space deep dive) articulated it clearly: go for *depth* — serial autoresearch loops, more thinking per task — rather than *breadth* (spinning up 50 parallel LLM runs). Separately, Dex Horthy, who coined "Context Engineering" and was one of the loudest vibe-coding advocates, publicly retracted his take and urged people to actually read the AI-generated code. The Shopify episode also surfaced a key shift: the real bottleneck in AI-assisted coding is no longer generation — it's review, CI/CD, and deployment stability.

**Why it matters:** Your audience is still in the "spin up more agents" phase. The people actually running AI at scale have already moved past it. "Tasteful Tokenmaxxing" is a concept that bridges both — it validates their instinct to use AI more, while redirecting them toward the thing that actually moves the needle.

### 2. Same model. Different scaffold. 4× better results.
**What it is:** A new benchmark study (surfaced heavily in AINews) put Qwen3.6-35B through the Polyglot coding benchmark using two different agent harnesses. With the default setup: 19%. With a different scaffold (little-coder agent): 78.7% — top 10 overall. Same model, same weights, one change. AINews commentary: "local models may underperform due to harness mismatches," not because the model is actually weak. This lands on the same day that Qwen3.6-27B dropped — a 27B dense model that beats the much larger Qwen3.5-397B-A17B on coding evals (SWE-bench Verified 77.2 vs 76.2), available locally via Unsloth in 18GB RAM.

**Why it matters:** This is the most actionable non-obvious insight in AI right now. Most people are still debating model selection when the scaffold is the bottleneck. And the corollary is immediate: you may already be using the right model — you just need a better harness. The Qwen3.6-27B local angle gives a practical First Step follow-on.

**Reel angle:** "Your model isn't the problem — same model, 4× better on the benchmark, one change. Here's what actually matters."

### 3. GitHub Copilot went model-agnostic — you can now use Claude in VS Code on any plan
**What it is:** GitHub Copilot rolled out bring-your-own-key/model support across all plans (individual, business, enterprise). You can now use Anthropic (Claude), Gemini, OpenAI, OpenRouter, Azure, Ollama, or any local backend — without switching your IDE or your Copilot subscription. VS Code becomes a neutral harness; the model is your choice. Russell Kaplan from Cognition made the complementary business case: enterprise buyers want model flexibility across the full SDLC, not lock-in to one lab.

**Why it matters:** Copilot has the largest installed base of any AI coding tool. For most devs, this is the lowest-friction path to trying Claude or a local model — no new tool, no new setup, just a model swap in a setting they already have. Classic First Step territory.

## Quick Scan

- **Google Cloud Next — TPU v8 split design:** 8t for training (~3× compute per pod vs Ironwood), 8i for inference (1,152 TPUs per pod, low latency). Google claims 1 million TPUs in a single training cluster. The productization story matters as much as hardware: Google is vertically integrating chips + models + agents + enterprise control planes.
- **Google Gemini Enterprise Agent Platform:** Vertex AI evolved into a platform for building, governing, and optimizing agents at scale (200+ models, Agent Studio, Workspace Intelligence GA). Enterprise agents are now a first-class Google product surface.
- **OpenAI Workspace Agents (Codex-powered):** Shared agents for Business/Enterprise/Edu/Teachers teams — operate across docs, email, chat, code, Slack, with scheduled and background tasks. OpenAI and Google are making the same enterprise move simultaneously.
- **OpenAI Privacy Filter (open-source):** Lightweight Apache 2.0 MoE (1.5B total / 50M active params, 128k context) for PII detection and redaction. Practically interesting for anyone building enterprise agent pipelines where cheap on-device PII masking matters before passing data to the LLM.
- **Qwen3.6-27B released (Apache 2.0):** Dense 27B with thinking + non-thinking modes, multimodal. Beats much larger 397B MoE on coding. Unsloth GGUFs run in 18GB RAM; Ollama and llama.cpp support same day. Early user reports strong on frontend/design/image.
- **Xiaomi MiMo-V2.5-Pro:** Claims SWE-bench Pro 57.2, Claw-Eval 63.8, 1,000+ autonomous tool calls. MiMo-V2.5 (non-Pro) adds 1M token context + omnimodality. The open agentic model competition is intensifying beyond Qwen.
- **Traces/ADP becoming the agent data primitive:** Convergence forming around the idea that traces → skills → context engineering → subagents → evals is the right loop for improving agents. LangChain, Clement Delangue (HuggingFace), and ADP (Agent Data Protocol) all pushing this.
- **Over-editing benchmark:** New study on coding models rewriting too much when fixing bugs. Opus 4.6 over-edits the least; GPT-5.4 over-edits the most. RL training outperforms SFT/DPO for learning surgical editing style. Practical for anyone who's noticed models "fixing" things that weren't broken.
- **Claude Code auto mode (Anthropic Eng):** Classifiers now auto-approve the ~93% of permission prompts users typically approve anyway. Meaningful friction reduction for long agentic sessions.
- **Shopify AI stack deep dive (Latent Space):** Tangle (reproducible ML experimentation), Tangent (automatic optimization), SimGym (customer behavior simulation) — what enterprise AI-native actually looks like at a $200B company. Unlimited Opus 4.6 internal token budget.
