### 1. The AI Coding Stack Nobody Planned — Cursor + Claude Code + Codex Are Now One Thing
**What it is:** Cursor 3 shipped on April 2 with a full parallel-agents window. OpenAI then published an official Codex plugin that runs *inside* Claude Code. By mid-April, early adopters were running all three together: Cursor for the IDE/agents layer, Claude Code for deep logic and multi-file reasoning, Codex for async fire-and-forget tasks.

**Why it matters:** The "which tool do I use?" question is over. The emerging answer is: you orchestrate all three. This mirrors how people run multiple SaaS tools but for your coding agent layer. The developer who figures out the division of labor first has a significant edge.

### 2. ChatGPT Workspace Agents — Custom GPTs Grew Up
**What it is:** OpenAI launched Workspace Agents on April 22 — team-shared agents powered by Codex that live inside ChatGPT or Slack, run 24/7 in the cloud, and handle repeatable workflows without anyone being logged in. You build it once; your whole org uses it. Integration with Slack means an agent can own a channel, respond to @mentions, and close tickets autonomously. Free until May 6, then credit-based pricing.

**Why it matters:** This is what custom GPTs were supposed to be. The shift: agents are no longer personal assistants — they're team infrastructure. A company can now deploy a "Lead Outreach Agent" or "Weekly Metrics Reporter" that runs like a headless employee. The May 6 pricing deadline is a real forcing function for teams to start now.

**Format:** First Step

**Reel angle:** *"OpenAI just gave every team a headless employee. Here's how to set one up before May 6 when it stops being free."*

### 3. MCP + A2A — The Two Protocols That Are Quietly Becoming the Plumbing of AI
**What it is:** Google's Agent2Agent (A2A) protocol — designed so agents from different companies can hand tasks to each other — hit production at 150 organizations at Google Cloud Next 2026, with Microsoft, AWS, Salesforce, SAP, and ServiceNow all running it. It's now governed by the Linux Foundation and versioned (v1.2). Meanwhile MCP is already the standard for how an agent connects to tools/data. The two now complement each other: MCP = agent ↔ tools; A2A = agent ↔ agent.

**Why it matters:** Most people using AI tools have no idea this layer exists — but it's what makes a Salesforce Agentforce agent hand off to a Google Vertex agent hand off to ServiceNow without any of them understanding each other's internals. This is the moment to explain it before it becomes assumed knowledge.

**Format:** Explainer

**Reel angle:** *"There are two new protocols everyone building with AI needs to know. One connects agents to tools. The other connects agents to agents. Here's the 60-second version."*
